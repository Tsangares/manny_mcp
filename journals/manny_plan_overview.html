The goal, in three levels

The confusion is usually here — three different goals get talked about at once. They're nested: each one exists to serve the one above it.

North star

An LLM that plays OSRS unattended ◆

A growing library of routines — YAML step-sequences built from atomic commands (MINE_ORE, BANK_OPEN, GOTO). An LLM supervisor watches routines run and, when no routine fits the situation, writes one on the spot. Point a fresh account at it → it grinds money-makers. There is no "finished" — the routines are the product.

This campaign

Make that vision buildable

Two things had to be true first. (1) The codebase had to stop being one 23,683-line file, so an LLM can author routines against clean, well-named commands instead of a swamp. (2) Prove the stack can run hands-off — Tutorial Island as the first unattended routine. You chose: finish the refactor first, then routines.

Right now

Finish the last slices of the split

Break the remaining handlers out of the giant file into focused support classes, in parallel, with the game client off to keep the machine cool. Two phases left, then a docs pass — then we pivot to routines.

The one hard constraint — heat

The crashes are hardware/power, not software. There's no code fix. So the whole workflow is shaped around it:

  • Commit every unit. A crash costs nothing — we resume from git, not from scratch.
  • Client off during code work. The game client pins a CPU core at ~79% on software rendering and cooks the package to 77°C+. It's only turned on for a live test, then killed.
  • Don't stack agents while the client runs. Agents + client together is the hot combination. Code lanes run while it's off (like now).
  • Parked option: move the client to a LAN machine (diort) that won't overheat — ends the crash cycle without the proxy headache. Revisit after the refactor.

What "done" unlocks

Tutorial Island already passes end-to-end when I drive it live — that's the acceptance test the whole refactor is measured against. Finishing the refactor flips us into the routines phase:

  1. Make Tutorial Island replay from one command (not hand-driven).
  2. Run grind-loop money-makers (woodcutting, combat training) unattended in Lumbridge.
  3. Start building the LLM supervisor that monitors routines and authors new ones on demand — the north star.
